# Read e2e test contract values from runtime-values instead of hardcoded CBOR

Replace hardcoded CBOR strings in e2e test config with dynamic reads from
contracts-info.json and plutus-local.json generated by the contract-compiler.
This ensures tests use the correct policy IDs and addresses after any change
to the local environment genesis transaction.
